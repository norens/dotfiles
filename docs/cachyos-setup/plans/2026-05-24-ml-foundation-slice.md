# ML Foundation Slice — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the storage layout and container runtime, then launch Ollama as an always-on Tailscale-exposed LLM service — reaching the milestone of "from MacBook, `curl http://cachyos:11434/api/tags` returns a model list".

**Architecture:** Container-first (Podman rootless + NVIDIA CDI). Two new btrfs subvols (`@ml-data`, `@containers`) carry bulk ML data outside snapper. Ollama runs as a `systemd --user` Podman Quadlet, exposed only on the Tailscale interface.

**Tech Stack:** btrfs subvols, Podman rootless, NVIDIA Container Toolkit (CDI), Ollama, Tailscale, chezmoi (for cross-OS `.zshrc` env), 1Password (for NGC API key storage in Plan 3).

**Source spec:** `docs/cachyos-setup/specs/2026-05-24-ml-robotics-stack-design.md`

## Task 1: Create `@ml-data` btrfs subvol

**Files:**
- Create on CachyOS: subvol `@ml-data` at btrfs root, plus `/home/nazar/ml-data` mount-point dir

**Context:** CachyOS default layout has `@`, `@home`, `@.snapshots`, `@var` subvols on the btrfs filesystem mounted at `/`. We add `@ml-data` at the same level so it's outside `@home` and can have its own mount opts (`nodatacow`, no compression).

- [ ] **Step 1: Identify the btrfs root device and find the top-level subvol mount**

Run on CachyOS:

```bash
findmnt -t btrfs
sudo btrfs subvolume list /
```

Expected: `findmnt` shows the device backing `/` (e.g. `/dev/nvme0n1p3`). `subvolume list` shows existing `@`, `@home`, `@.snapshots`, `@var` with their IDs.

Record the device path for next steps.

- [ ] **Step 2: Mount the btrfs top-level (subvolid=5) temporarily**

```bash
sudo mkdir -p /mnt/btrfs-root
sudo mount -o subvolid=5,noatime <DEVICE> /mnt/btrfs-root
ls /mnt/btrfs-root
```

Replace `<DEVICE>` with the device from Step 1.

Expected: directory listing shows `@`, `@home`, `@.snapshots`, `@var` as directories (the existing subvols).

- [ ] **Step 3: Create the `@ml-data` subvol at the top level**

```bash
sudo btrfs subvolume create /mnt/btrfs-root/@ml-data
sudo btrfs subvolume list /
```

Expected: `subvolume list` now includes `@ml-data` with a fresh ID.

- [ ] **Step 4: Create the mount-point directory and unmount the top-level**

```bash
sudo mkdir -p /home/nazar/ml-data
sudo umount /mnt/btrfs-root
sudo rmdir /mnt/btrfs-root
```

Expected: no errors. `/home/nazar/ml-data` exists but is empty (will mount in Task 3).

- [ ] **Step 5: Commit (no repo changes yet, this is system state)**

Nothing to commit in git. Move to Task 2.

```bash

sudo umount /mnt/btrfs-root
```

## Task 3: Add fstab entries and mount both subvols

**Files:**
- Modify on CachyOS: `/etc/fstab`

**Context:** Both subvols need persistent mount entries with the correct opts: `nodatacow`, `noatime`, no compression, btrfs metadata for the subvol name.

- [ ] **Step 1: Get the UUID of the btrfs filesystem**

```bash
sudo blkid <DEVICE> | grep -oP 'UUID="\K[^"]+'
```

Record the UUID (e.g. `5f4c8a3b-1234-5678-9abc-def012345678`).

- [ ] **Step 2: Back up the current fstab**

```bash
sudo cp /etc/fstab /etc/fstab.bak-pre-ml-subvols
```

Expected: `/etc/fstab.bak-pre-ml-subvols` exists.

- [ ] **Step 3: Append the two new fstab entries**

Edit `/etc/fstab` with `sudoedit /etc/fstab`. Append these two lines at the end (replace `<UUID>` with the UUID from Step 1):

```
UUID=<UUID>  /home/nazar/ml-data    btrfs  rw,noatime,nodatacow,ssd,discard=async,subvol=@ml-data    0 0
UUID=<UUID>  /var/lib/containers    btrfs  rw,noatime,nodatacow,ssd,discard=async,subvol=@containers 0 0
```

- [ ] **Step 4: Validate fstab syntax**

```bash
sudo findmnt --verify --verbose
```

Expected: exits 0, no "I/E" lines reported. If errors — fix the fstab edit before mounting.

- [ ] **Step 5: Mount both new subvols**

```bash
sudo mount /home/nazar/ml-data
sudo mount /var/lib/containers
findmnt /home/nazar/ml-data
findmnt /var/lib/containers
```

Expected: `findmnt` shows both mounted with the expected opts including `nodatacow`.

- [ ] **Step 6: Fix ownership of `~/ml-data`**

```bash
sudo chown nazar:nazar /home/nazar/ml-data
ls -ld /home/nazar/ml-data
```

Expected: owner is `nazar:nazar`, drwxr-xr-x.

- [ ] **Step 7: Create the subdirectory layout**

```bash
mkdir -p ~/ml-data/{hf-cache,isaac-cache,datasets,checkpoints,logs}
ls -la ~/ml-data
```

Expected: five subdirs visible, all owned by `nazar:nazar`.

Optionally, take a snapper snapshot of `/` to mark the state:


## Task 4: Wire `HF_HOME` into shell via chezmoi

**Files:**
- Modify in chezmoi repo: `dot_zshrc` (add Linux-conditional block)

**Context:** PyTorch / HuggingFace libraries default to `~/.cache/huggingface` for model storage. We need to redirect this to the new `~/ml-data/hf-cache` subvol via `HF_HOME` env var. macOS doesn't need this (no separate ML subvol there) — use a chezmoi templating condition.

- [ ] **Step 1: Check whether `dot_zshrc` already has a Linux block**

On the machine where you edit chezmoi source (MacBook is fine):

```bash
cd ~/.local/share/chezmoi
grep -n 'chezmoi.os' dot_zshrc | head -20
```

Expected: shows existing conditional blocks if any. If `dot_zshrc` is not a `.tmpl` file, you must rename it first:

```bash
chezmoi chattr +template ~/.zshrc
git add -A && git status
```

(That converts `dot_zshrc` → `dot_zshrc.tmpl`.)

- [ ] **Step 2: Add the Linux-only ML env block**

Open `dot_zshrc.tmpl` (now templated) and add this block. Recommended placement: near the bottom of the file, after PATH but before tool inits like `eval "$(starship init zsh)"`. The block:

```sh
{{ if eq .chezmoi.os "linux" -}}
# --- ML / HuggingFace cache redirection (Linux only) ---
# See docs/cachyos-setup/specs/2026-05-24-ml-robotics-stack-design.md §2
export HF_HOME="$HOME/ml-data/hf-cache"
export TRANSFORMERS_CACHE="$HF_HOME/hub"
export HF_DATASETS_CACHE="$HF_HOME/datasets"

# Podman rootless: explicit userns for predictable UID mapping
export PODMAN_USERNS="keep-id"
{{ end -}}
```

- [ ] **Step 3: Verify chezmoi sees the template change cleanly**

On MacBook:

```bash
chezmoi diff ~/.zshrc
```

Expected: shows no diff on macOS (because the conditional is `linux`-only). If you see additions to your macOS `.zshrc`, the template condition is wrong — fix the `{{ if }}` syntax.

- [ ] **Step 4: Commit the chezmoi change**

```bash
cd ~/.local/share/chezmoi
git add dot_zshrc.tmpl
git commit -m "feat(zsh): add Linux HF_HOME pointing to ml-data subvol"
git push
```

- [ ] **Step 5: Apply on CachyOS**

On CachyOS:

```bash
chezmoi update
echo "HF_HOME=$HF_HOME"
```

Expected: after `chezmoi update`, opening a new shell shows `HF_HOME=/home/nazar/ml-data/hf-cache`.

- [ ] **Step 6: Verify by writing a marker file**


## Task 6: Define Ollama Quadlet unit

**Files:**
- Create in chezmoi: `dot_config/containers/systemd/ollama.container.tmpl`
- (After chezmoi apply) Auto-generated on CachyOS: `~/.config/containers/systemd/ollama.container`

**Context:** Quadlets are systemd-native Podman unit definitions (introduced in Podman 4.4). User-level Quadlets at `~/.config/containers/systemd/*.container` are picked up by `systemd --user` and translated into `*.service` units at runtime. Ollama is a great fit: stateful (model storage), long-lived, single instance.

- [ ] **Step 1: Verify the Quadlet path on CachyOS**

```bash
systemctl --user status
ls ~/.config/containers/systemd/ 2>/dev/null || echo "(dir not yet created)"
```

Expected: `systemctl --user` works (no errors); the Quadlet dir may not exist yet.

- [ ] **Step 2: Determine Tailscale interface IP on CachyOS**

```bash
tailscale ip -4
ip -4 addr show dev tailscale0
```

Record the IPv4 (e.g. `100.x.y.z`). We'll bind Ollama to this exact address.

- [ ] **Step 3: Create the Quadlet template in chezmoi**

On MacBook (in chezmoi source dir), create the file:

```bash
mkdir -p ~/.local/share/chezmoi/dot_config/containers/systemd
```

Create `~/.local/share/chezmoi/dot_config/containers/systemd/ollama.container.tmpl` with this content:

```ini
{{- /* Render only on Linux (Quadlets are systemd, macOS doesn't need this) */ -}}
{{- if eq .chezmoi.os "linux" -}}
# Ollama always-on LLM container.
# Managed by chezmoi; do not edit the rendered file directly.
# Source spec: docs/cachyos-setup/specs/2026-05-24-ml-robotics-stack-design.md §5

[Unit]
Description=Ollama LLM service (always-on, Tailscale-only)
After=network-online.target tailscaled.service
Wants=network-online.target

[Container]
ContainerName=ollama
Image=docker.io/ollama/ollama:latest
AutoUpdate=registry

# GPU via CDI
AddDevice=nvidia.com/gpu=all

# Bind to Tailscale interface only (replace 100.x.y.z with `tailscale ip -4` output)
PublishPort={{ .tailscaleIP }}:11434:11434

# Persistent model storage on the @ml-data subvol
Volume=%h/ml-data/ollama-models:/root/.ollama:Z

# Health
HealthCmd=ollama --version
HealthInterval=30s
HealthRetries=3

# Resource hints (loose; let workloads use most of GPU)
Environment=OLLAMA_KEEP_ALIVE=10m
Environment=OLLAMA_NUM_PARALLEL=2

[Service]
Restart=on-failure
RestartSec=10

[Install]
WantedBy=default.target
{{- end -}}
```

- [ ] **Step 4: Add the Tailscale IP to chezmoi data**

The template uses `{{ .tailscaleIP }}` — chezmoi looks this up from per-host data. On CachyOS, create the host-specific data file:

```bash
mkdir -p ~/.config/chezmoi
cat > ~/.config/chezmoi/chezmoi.toml <<EOF
[data]
tailscaleIP = "$(tailscale ip -4)"
EOF
cat ~/.config/chezmoi/chezmoi.toml
```

Expected: file contains `tailscaleIP = "100.x.y.z"`.

This file is NOT tracked in the chezmoi repo (it's per-machine config). Verify:

```bash
chezmoi data | grep tailscaleIP
```

Expected: shows your tailscale IP.

- [ ] **Step 5: Verify the template renders correctly (dry-run on CachyOS)**

```bash
chezmoi execute-template < ~/.local/share/chezmoi/dot_config/containers/systemd/ollama.container.tmpl
```

Expected: prints the Quadlet INI with `PublishPort=100.x.y.z:11434:11434` filled in. If `{{ .tailscaleIP }}` appears unsubstituted — chezmoi data isn't loading; fix `chezmoi.toml` location.

- [ ] **Step 6: Commit the chezmoi template**

On MacBook:

```bash
cd ~/.local/share/chezmoi
git add dot_config/containers/systemd/ollama.container.tmpl
git commit -m "feat(cachyos): add Ollama Quadlet template (Tailscale-only)"
git push
```
