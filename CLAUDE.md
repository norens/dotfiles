# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a **bare-repo dotfiles** setup for macOS. The git directory lives at `~/macos-configs` (bare) while the work tree is `$HOME`. This means config files live at their real paths (e.g. `~/.zshrc`, `~/.config/karabiner/karabiner.json`) and are tracked directly.

All git operations use the `cfg` alias:
```bash
alias cfg='git --git-dir=$HOME/macos-configs --work-tree=$HOME'
```

The `.gitignore` uses an **allowlist pattern**: ignore everything (`*`), then selectively un-ignore tracked files with `!` entries.

## Repository Structure

### Shell
- `~/.zshrc` - Zsh config (PATH, plugins, tool init)
- `~/.zsh_aliases` - Shell aliases (includes `cfg` alias)
- `~/.gitconfig` - Git user config

### Window Management
- `~/.config/karabiner/karabiner.json` - Karabiner-Elements config (generated by Goku)
- `~/.config/goku/karabiner.edn` - Goku source (edit this, not karabiner.json directly)
- `~/.config/skhd/skhdrc` - skhd hotkey daemon config
- `~/.config/yabai/yabairc` - yabai tiling window manager config
- `~/.config/ymsp/ymsp.config.json` - YMSP (yabai move/swap/place) config

### Tools
- `~/.config/brewfile/Brewfile` - Homebrew packages, casks, and Mac App Store apps
- `~/.config/gh/config.yml` - GitHub CLI config (NOT hosts.yml - contains auth)
- `~/.ssh/config` - SSH config (NOT keys)
- `~/.config/thefuck/settings.py` - thefuck settings
- `~/.config/git/ignore` - Global gitignore

### IDE
- `~/Library/Application Support/Cursor/User/keybindings.json` - Cursor keybindings
- `~/Library/Application Support/Cursor/User/settings.json` - Cursor settings
- `~/.config/jetbrains-keymaps/macOS_mod.xml` - Canonical JetBrains keymap (distributed to all IDEs via setup.sh)

### macOS
- `~/Library/Preferences/com.apple.HIToolbox.plist` - Input sources config

### Scripts
- `~/scripts/merge_pr.sh` - Interactive PR merge helper using `gh` CLI
- `~/scripts/script_manager.sh` - CLI for managing shell aliases and scripts
- `~/scripts/setup.sh` - Bootstrap script for new Mac setup

### Repo Meta
- `~/.gitignore` - Allowlist gitignore (lives at $HOME root)
- `~/CLAUDE.md` - This file

## Key Conventions

- **Bare repo**: Always use `cfg` instead of `git` for dotfiles operations (`cfg add`, `cfg commit`, `cfg push`, etc.)
- **Allowlist gitignore**: When adding new config files, add corresponding `!` entries to `~/.gitignore`. For nested paths, each parent directory needs its own `!` entry.
- **Karabiner via Goku**: Edit `~/.config/goku/karabiner.edn`, then run `goku` to regenerate `karabiner.json`. Never edit `karabiner.json` directly.
- **JetBrains keymaps**: Edit `~/.config/jetbrains-keymaps/macOS_mod.xml` as the single source of truth. Run `scripts/setup.sh` (section 5) or manually copy to `~/Library/Application Support/JetBrains/*/keymaps/`.
- **Shell scripts**: Use `#!/bin/bash`, include input validation, use `gh` CLI for GitHub operations.
- **No secrets**: Never track auth tokens, SSH keys, or API keys. Check the "Security" section in the plan if unsure about a file.
- **No build system**: This is a config-only repo with no build, lint, or test commands.

## Common Operations

```bash
# Check what's changed
cfg status
cfg diff

# Add a new config file
cfg add ~/.config/newapp/config.yml
# Then add !.config/newapp and !.config/newapp/config.yml to ~/.gitignore

# Commit and push
cfg add -u  # stage all tracked changes
cfg commit -m "description"
cfg push

# Update Brewfile after installing new packages
brew bundle dump --file=~/.config/brewfile/Brewfile --force
```
